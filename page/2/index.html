懋德
一五年 五月

有年冬天一家旧货店，我用手机从一个铜质的墨盒盖上拍下这几句话；偶然翻出来看一看，看了几年；最后给了绍昆也看，他也喜欢，一拍即合；曲经三易，写成这一首歌。
一九二六，北伐兵兴，边隅之地，青年两个，墨盒为礼，寄语未来；惟不知抬头落款中的两个人，他们的后来，是否就达学问功名，是否完成…『伟大的人生』。
我没有为之想得一个适当的标题，只好自作聪明叫做『晚春』这两个字。仅为对应落款的时序；又与小津片名相仿，也不避。
藉此，作腰乐队完结终曲。」
— 腰乐队 刘弢

晚春 腰乐队(Kidney) 作词：刘涛 作曲：杨绍昆 吉他/键盘：杨绍昆 贝斯：胡尚洪 鼓：贺正超 哥哥你今回的北游 觉悟了生命的充实 领略了友情的真挚 社会阵场上的勇将 在轰烈的炮火中间 别忘却身心的和睦 奋勇呀然后休息呀 完成你伟大的人生

偶然发现我对表达了这样内核的文艺作品都难掩喜爱，亦或是田村卡夫卡那个顽强的十五岁少年,亦或是Lee Chandler那句”I Can’t Beat it”，亦或是这里的”奋勇呀然后休息呀,完成你伟大的人生”。
坦白来说大部分时间，我不会在意一首歌的歌词，我理解歌词本身在因音乐的作用本就是可有可无的一小部分。但是仍然有几首歌的歌词让我触动，粗糙的编曲和真挚的情感。

夏与冬的奏鸣曲

发表于 2018-11-29 更新于 2022-11-05 分类于 Reading

从开始的灰度风格，到最后随着立体主义的展开、现实随之崩坏。一个关于以和音为宗教的乌托邦的故事开始的诡谲推理小说。《夏与冬的奏鸣曲》核心应该就是立体主义的“展开”和“实体化”。”将’核心’实体化是从虚无的空间中解脱出来的唯一办法”，感觉夏与冬的奏鸣曲其实更像是书中所说的《启示录》，是春与秋的奏鸣曲的后续，也是将和音实体化的后续。二十年前七个人将自己最认同的“自我”部分，投射到“和音”之中，奉献给和音，期望通过一系列的“还原”和“展开”创造出自己的神灵，只是对真宫和音展开最终失败，导致二十年后桐璃的到来完成“展开”的使命，而桐璃的左眼应该就是完成和音展开的最后一个部分。
“他认定的桐璃，一口喝干咖啡。曾经美丽的黄色双眸，闪闪发光。”而最后乌有带走的竟然是双眼完好桐璃、他认定的桐璃，猜想这个大约就是完成实体化的和音吧。
PS : 关于立体主义“展开”和随之而来的崩坏的感觉竟然和当初接触《沙耶之歌》地感觉极其相似

Hexo框架下的博客提交至Google/Bing搜索

将 Hexo 博客提交至搜索引擎的核心流程分为三步：

1 验证网站所有权：向 Google 和 Bing 证明您是该网站的所有者。

2 生成并提交站点地图 (Sitemap)：提供一个包含您网站所有链接的地图，方便搜索引擎全面抓取。

3 配置 Robots.txt (可选但推荐)：指导搜索引擎哪些内容可以抓取，哪些需要忽略。

先以Google为例

第一步 验证网站所有权

此步骤需要在 Google Search Console 和 Bing Webmaster Tools 中分别操作，但方法类似。推荐使用 HTML 文件验证法，因为它只需一次部署，即可被多个搜索引擎使用。

1 登录平台

  • Google: 前往 Google Search Console 并登录您的 Google 账号。

  • Bing: 前往 Bing Webmaster Tools 并登录您的 Microsoft 账号。

2 添加博客网址添加属性，

  • 在各自的平台中，选择“添加属性”或“添加网站”，然后将博客地址https://yourname.github.io/添至相应位置。

3 谷歌搜索有多种验证方式，这里我选择谷歌推荐的选择 HTML 文件验证方式。

  • 选择 HTML 文件验证 方式。

  • 从 Google 下载 HTML 验证文件。

  • 将这两个文件复制到您博客项目的 根目录Root directory/sources/ 文件夹下。

  • 配置配置文件使 hexo g 跳过对该文件的渲染：找到 skip_render 配置项，将两个验证文件的文件名添加进去。如果 skip_render 不存在，请手动添加。

4 部署并完成验证:hexo三连hexo clean && hexo generate && hexo deploy
